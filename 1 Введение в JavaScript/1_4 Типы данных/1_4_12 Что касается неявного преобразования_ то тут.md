Что касается неявного преобразования, то тут все просто - функции, которые ожидают на вход какой-то определенный тип данных, пытаются автоматически привести полученные данные к необходимому типу. Приведение работает по тем же правилам, по которым работают функции явного приведения - String(), Number() и Boolean(). 

Приведем пару примеров.

В модуле MATH (набор математических функций для JavaScript, его мы подробно рассмотрим в следующем модуле) есть функция cos() - вычисление косинуса.
Если мы в нее передадим значение 0, то резульат работы функции будет ожидаемо - 1.

let x = 0;  
console.log(MATH.cos(x));  // Результатом вывода в консоль будет число 1
В случае, если мы попытаемся передать в эту функцию не числовое, а, например, логическое значение, то оно будет автоматически приведено к числовому по тем же правилам, по которым работает функция Number(). Например, если мы попробуем посчитать значение косинуса для false, то оно будет автоматически приведено к цифровому значению, т.е. к нулю, и функция вычисления косинуса также вернет нам значение 1. 

let x = false;  
console.log(MATH.cos(x));  // Результатом вывода в консоль будет число 1
Главный вывод, который мы можем запомнить - неявные преобразования выполняются по тем же правилам, что и явные, в функциях Number(), String() и Boolean().
